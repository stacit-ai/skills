# Reproducible Training And Evaluation Projects

Use this branch for ML projects whose training and evaluation should be repeatable and
maintainable, but which are not unattended production services.

## Structure

Run `uv init` first, then copy the harness/tool files from
[assets/reproducible-training-template](../assets/reproducible-training-template).
The asset bundle intentionally contains no `pyproject.toml`; keep the file generated by
`uv init` and merge the tool configuration snippet below into it.

Expected shape:

- `AGENTS.md`
- `.agents/knowledge/WORKFLOW.md`
- `.agents/knowledge/QUALITY.md`
- `.agents/knowledge/REFERENCES.md`
- `pyproject.toml`
- `justfile`
- `.pre-commit-config.yaml`, when Git hooks are selected and composed for this project
- `configs/`, when needed
- `data/raw/`, `data/interim/`, `data/processed/`, when needed
- `outputs/`, when needed
- `scripts/`, when command recipes become too large
- `docs/`, when documentation is needed
- `notebooks/`, when human exploration is needed
- `<package_name>/`, created by or after `uv init`
- `tests/`, when targeted lightweight tests are needed

Do not create every optional directory if the project clearly does not need it, but keep
the data layering and harness files when reproducibility is a goal.

## Dependency And Command Defaults

- Use `uv` with the `pyproject.toml` generated by `uv init`.
- Centralize dependencies and tool configuration in `pyproject.toml`.
- Use the `.gitignore` generated by `uv init`; append project-specific ignored outputs
  such as run artifacts, processed data, or checkpoints when those directories are
  created.
- When PyTorch is used, apply [references/pytorch.md](pytorch.md): define the supported
  environment matrix and use mutually exclusive backend extras when similar machines
  need selectable CPU, CUDA, ROCm, or XPU builds.
- Use `justfile` as the main command surface. Default target prints command help.
- Include commands for setup, dependency sync, formatting/checks, lightweight tests,
  manual slow tests, training, evaluation, and cleaning ignored outputs.

## Pre-Commit Composition

Generate `.pre-commit-config.yaml` from the selected project tools; do not copy a
universal config. Order non-security hooks by nearest, cheapest failure, then run
secret scanning last:

1. Basic file rules: trailing whitespace, EOF, YAML/TOML syntax where supported, merge
   conflicts, large files, and line endings.
2. `uv-lock` so changes to `pyproject.toml` cannot leave `uv.lock` stale.
3. Ruff formatting, then Ruff linting.
4. Type checking only when the project has enabled and configured it.
5. A stable, fast test subset only when it consistently completes quickly.
6. One secret scanner, Gitleaks by default; copy its hook block from
   [references/secret-scanning.md](secret-scanning.md) and keep it last.

Keep training, GPU, model downloads, full suites, and slow tests out of pre-commit.
Expose format, lint, type-check, fast-test, and aggregate check recipes as direct tool
commands in `justfile`; the aggregate recipe must not invoke pre-commit.

## Harness Rules

Use progressive harness loading:

- `AGENTS.md` maps the repo and tells agents when to read each knowledge file.
- `.agents/knowledge/WORKFLOW.md` stores training, evaluation, data preparation, and
  manual slow-test workflows.
- `.agents/knowledge/QUALITY.md` stores code quality, documentation, test, data, output,
  and git safety rules.
- `.agents/knowledge/REFERENCES.md` stores framework and tool documentation links.

Every lasting instruction needed by future agents must be in these files. Do not leave
important architecture rules only in chat or in the scaffolding skill context.

## Code Rules

- Put logical code under `<package_name>/`.
- Put workflow modules under a dedicated package submodule such as
  `<package_name>/workflows/`, and expose them through `justfile` commands.
- Use type annotations for readability and IDE support, but do not enable type checking
  by default.
- Public or cross-module functions/classes need docstrings with parameters, returns,
  and errors. Internal helpers can use concise purpose docstrings.
- Prefer readability over eliminating every repetition.
- Extract shared components for logical consistency, not merely because lines repeat.
- Fail early for unexpected states; handle dirty records only when the reason is known.

## Data, Config, Outputs, Tests

- `data/raw/` is immutable raw input. Only deliberate data-preparation commands write
  outside it.
- `data/interim/` stores intermediate artifacts shared between workflow steps.
- `data/processed/` stores final data products such as datasets or predictions.
- `outputs/` stores logs, checkpoints, and run artifacts and is ignored by git. Code
  must not depend on it already existing.
- Prefer coarse experiment configuration. Use Hydra only when composition is genuinely
  helpful; otherwise use a simpler TOML and settings approach.
- Do not mirror every code module into config. Config values should represent
  experiment choices such as data source, base model, hyperparameters, and output paths.
- Keep CI and any generated pre-commit configuration light: formatting/lint checks and
  fast tests only. Expensive smoke tests and training runs are manual commands.

## Pyproject Tool Configuration

After `uv init`, add development dependencies:

```sh
uv add --dev pre-commit ruff ty pytest pytest-cov pytest-mock
```

Then merge these sections into the generated `pyproject.toml`. Do not replace the
generated `[project]` metadata.

```toml
[tool.ruff]
line-length = 120

[tool.ruff.format]
docstring-code-format = true
docstring-code-line-length = 80

[tool.ruff.lint]
extend-select = [
    "I",    # isort: Import sorting
    "N",    # pep8-naming: Check PEP 8 naming conventions
    "TID",  # flake8-tidy-imports: Import policy checks
    "W",    # pycodestyle warning: PEP 8 styling warnings
]
unfixable = [
    "F401", # unused-import: Unused import
]

[tool.ruff.lint.per-file-ignores]
"scripts/*" = [
    "TID",  # flake8-tidy-imports: Tidy up imports
]
"tests/**/*" = [
    "TID252",   # relative-imports: Relative imports ban
]

[tool.ruff.lint.flake8-tidy-imports]
ban-relative-imports = "all"

[tool.pytest.ini_options]
minversion = "6.0"
addopts = "-ra -q"
testpaths = ["tests"]
markers = [
    "slow: manual-only tests that may require GPU, model downloads, or long runs",
]

[tool.ty.src]
include = ["."]
exclude = [
    "scripts",
    "notebooks",
]
```
