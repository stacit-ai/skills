# Gitmoji Reference

Source: https://gitmoji.dev/

All official gitmoji entries. Format for each: emoji, text code, name, description.

## Code Quality & Structure

| Emoji | Code | Name | When to use |
|---|---|---|---|
| 🎨 | `:art:` | art | Improve structure or format of the code |
| ♻️ | `:recycle:` | recycle | Refactor code (no behavior change) |
| 💡 | `:bulb:` | bulb | Add or update comments in source code |
| 💩 | `:poop:` | poop | Write bad code that needs to be improved later |
| 🧵 | `:thread:` | thread | Add or update code related to multithreading or concurrency |
| 🦺 | `:safety_vest:` | safety_vest | Add or update code related to validation |

## Disambiguation: Similar Emojis

| Situation | Use | Not |
|---|---|---|
| New user feature | ✨ `:sparkles:` | 🎉 `:tada:` (only for project init) |
| Code bug fix | 🐛 `:bug:` | 🩹 `:adhesive_bandage:` (for trivial, non-critical fixes) |
| Security vulnerability fix | 🔒️ `:lock:` | 🐛 `:bug:` |
| Remove a file or module | 🔥 `:fire:` | 🗑️ `:wastebasket:` (for deprecated, not yet deleted) |
| Performance improvement | ⚡️ `:zap:` | ♻️ `:recycle:` (for restructuring without perf focus) |
| Upgrade a dependency | ⬆️ `:arrow_up:` | 📦️ `:package:` (for compiled output, not deps) |
| Fix a flaky or broken test | ✅ `:white_check_mark:` | 🧪 `:test_tube:` (for writing a failing test to reproduce a bug) |
